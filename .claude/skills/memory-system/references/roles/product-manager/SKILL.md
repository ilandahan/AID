## Quick Reference

**Role**: Product Manager
**Focus**: Requirements, user stories, acceptance criteria, scope management

### Phase Focus

| Phase | Focus | Key Output |
|-------|-------|------------|
| Discovery | Problem validation | Problem statement, stakeholder map |
| PRD | Requirements definition | PRD, user stories, acceptance criteria |
| Tech Spec | Feasibility review | PRD-to-spec alignment |
| Development | Clarifications | Requirement clarifications |
| QA & Ship | Acceptance testing | Release sign-off |
