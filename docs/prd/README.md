# PRD (Product Requirements Documents)

Store all Product Requirement Documents in this folder.

## Naming Convention

```
YYYY-MM-DD-feature-name.md
```

Example: `2024-06-15-user-authentication.md`

## Template

Each PRD should include:

1. **Overview** - What problem are we solving?
2. **Goals** - What are we trying to achieve?
3. **User Stories** - Who uses this and how?
4. **Requirements** - Functional and non-functional
5. **Success Metrics** - How do we measure success?
6. **Out of Scope** - What we're NOT doing

## Usage

When writing tests, ALWAYS reference the latest PRD for the feature to ensure:
- All user stories have test coverage
- Requirements are validated
- Edge cases from requirements are tested
